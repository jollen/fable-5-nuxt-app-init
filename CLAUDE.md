# Project Memory

## Project Goal
Build a minimal Nuxt Fullstack News Reader.
The backend provides a semiconductor news analysis REST API.
The frontend renders a readable news reader page.

## Commands
- Install dependencies with npm install.
- Run npm run build after changing Nuxt pages, components, or server routes.
- Run git diff --stat before preparing the final summary.

## Coding Rules
- Use ES module syntax for new JavaScript files.
- Use Vue Composition API for new Vue components.
- Do not add external UI libraries unless the task explicitly allows it.
- Keep the API response schema stable unless the task explicitly changes it.

## Architecture
- Server API routes live in server/api/.
- Mock news data lives in server/data/.
- Reusable UI components live in components/.
- The main reader page lives in app.vue.

## Workflow
- Read related files before editing.
- Create a short plan before implementation.
- Modify only files related to the current task.
- Report changed files, validation commands, and remaining risks.
